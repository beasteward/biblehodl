# ⚡ BibleHodl

A self-hostable community platform built on [Nostr](https://nostr.com). Each community owns their entire stack — relay, file storage, database, and frontend. Designed for small groups (50–300 users) who want digital sovereignty.

## Stack

| Service | Tech | Purpose |
|---------|------|---------|
| **App** | Next.js 16, React 19, Prisma, Tailwind 4 | Frontend + API |
| **Relay** | nostr-rs-relay (Rust/SQLite) | Chat, DMs, calendar, meetings |
| **Files** | Blossom (Deno) | Media storage via Nostr-native protocol |
| **Proxy** | Caddy | Reverse proxy, automatic HTTPS |
| **Bible API** | CPDV Bible (shared) | Scripture lookups (optional) |

Everything runs in Docker Compose on a single VPS.

## Features

- **Teams** — Create and manage community groups
- **Chat** — Nostr NIP-28 channel messaging
- **DMs** — Encrypted direct messages (NIP-17)
- **Calendar** — Community events (NIP-52)
- **Meetings** — Schedule and manage group sessions
- **Files** — Upload and share via Blossom
- **Games** — Trivia and interactive sessions
- **Bible** — CPDV scripture integration

## Requirements

- VPS with 2+ vCPU, 4GB+ RAM (Ubuntu/Debian recommended)
- Docker & Docker Compose
- A domain with DNS pointing to your VPS:
  - `yourdomain.com` → App
  - `relay.yourdomain.com` → Nostr relay
  - `files.yourdomain.com` → Blossom file server

## Quick Start

### 1. Clone

```bash
git clone https://github.com/beasteward/biblehodl.git
cd biblehodl
git submodule update --init --recursive
```

### 2. Install dependencies (for init script)

```bash
npm install
```

### 3. Run the init script

```bash
./scripts/init.sh
```

This interactive script will:

- Collect your community name, domain, and branding
- Set up the admin account (generate or import a Nostr keypair)
- Generate all secrets and write your `.env`
- Bootstrap the database with the admin user and 5 invite codes
- Display credentials **once** — save them immediately

### 4. Start services

```bash
docker compose up -d
```

### 5. Visit your site

Open `https://yourdomain.com` — the admin can log in with their nsec (no invite code needed). Share the generated invite codes with your first members.

## Access Control

1. **Admin runs init script** → becomes the owner
2. **Admin creates invite codes** via the admin panel
3. **New members visit `/join`** → enter invite code + create/import Nostr keypair
4. **Registered pubkeys are whitelisted** on the relay automatically

Only whitelisted users can read/write to the community relay.

## Configuration

All config lives in `.env` (generated by `init.sh`). Key variables:

| Variable | Description |
|----------|-------------|
| `DOMAIN` | Your community's domain |
| `COMMUNITY_NAME` | Display name |
| `PRIMARY_COLOR` | Brand color (hex) |
| `ADMIN_NPUB` | Admin's Nostr public key |
| `NEXT_PUBLIC_RELAY_URL` | WebSocket relay URL |
| `NEXT_PUBLIC_BLOSSOM_URL` | Blossom file server URL |
| `SESSION_SECRET` | App session encryption key |

See `.env.example` for the full list.

## Self-Hosting Philosophy

Each BibleHodl instance is **fully independent**:

- Own database (SQLite)
- Own Nostr relay (private, whitelisted)
- Own file storage (Blossom)
- Own domain and TLS certificates
- No federation, no shared infrastructure, no back doors

The deployer runs the init script, hands credentials to the community admin, and walks away. No persistent privileged access.
